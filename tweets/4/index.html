[ asptweets ] Keeping track of ASP on Twitter, so you don't have to.

My name is Cameron, and I'm a full-time hospital Antimicrobial Stewardship Pharmacist and part-time computer geek from Ottawa, Canada.

I tried to start using Twitter to improve my ASP practice - it's a great way to keep up with the latest ASP/ID research and discussions. However, there's a massive volume of (often unrelated) content, and it's easy to get overwhelmed.

To that end, I created this site. It does its best to track tweets related to ASP and ID and scores them based on a custom algorithm. Every day, it automatically updates to show tweets from days prior that are considered "probably useful", with the most important ones at the top. It also gives a top 50 for the past week, if you just want a fast glance. The algorithm takes into account relevant keywords and leverages the power of social media, factoring in Favourites and Retweets to figure out what is most important.

I personally use the site daily, and will be active trying to fix glitches and improve the search algorithms. This also allows me to update the search parameters to include things like ongoing ID conference hashtags.

Special thanks to Brad Langford (@BRxAD) for giving the original conference presentation that got me into Twitter in the first place, as well as for his help and support (and use of his "ASP Tweeters" list) while developing the site.

I hope you will find this site useful, and feel free to send any feedback to me at @campicillin or cameron@asptweets.com

Details for Geeks
  • Programmed in Python 2
  • All scripts run automatically on a (mostly) always-on Raspberry Pi using cron
  • Tweepy used to scrape Twitter API
    • Search run every 15 minutes to get most recent tweets for variety of search terms
  • Tweets saved into local SQLite database using Peewee ORM
  • Flask used to build website
  • Frozen-Flask used to generic static files
    • Generated at 4am daily
  • Tweet embedding code grabbed direct from twitter GET API
  • Previews & Embedding HTML cached in database
  • Hosted on Github Pages
