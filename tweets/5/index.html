[ asptweets ] Keeping track of Twitter, so you don't have to.

My name is Cameron, and I'm a full-time hospital Antimicrobial Stewardship Pharmacist and part-time computer geek from Ottawa, Canada.

I recently attended an excellent talk from @BRxAD, entitled 3 Ways Twitter Improved my ID/ASP Practice. Inspired, I started trying to use Twitter for 3 main purposes: finding newly released and relevant ASP/ID literature, seeing what people were Tweeting about ASP related content, and seeing what kind of things the public were saying about broad topics like "antibiotics".

Quickly, I was overwhelmed the sheer volume of information coming my way, and was finding it impossible to keep up.

I knew the information I wanted was there, but there was no way I was going to get it just trying to follow Twitter on my own.

Thus, asptweets.com was born. It tracks Twitter under broad search terms relating to ASP and ID, and sorts through them daily to generate a list that is quickly consumable.

Pick a category at the top (e.g. "#antimicrobialstewardship") to see tweets related to it. For each category, you get the:

  • Top Links: shows things that were linked to from tweets, sorted by the top # of times that link was mentioned. This is great for finding new literature and interesting articles.
  • Most Popular: shows the top tweets as ordered by combined "retweeted" and "favourited" counts. See what people are talking about!

You can also see tweets from any of the past seven days, or aggregated for the entirety of the past week. Just click the "When do you want to view tweets from?" dropdown at the top. Great for when you've missed checking for a day or two!

Note that the site is fully automated and the algorithms aren't opinionated... so you might see things that are weird, unrelated, or generally not good information. The site is not endorsing anything posted on it, it's just a snapshot of crowdsourced interests. Also note that there will be some unavoidable duplication of content.

I personally use the site daily, and will be active trying to fix glitches and improve the search algorithms. I also keep a separate database of popular hashtags, so I can see if there's any other search headings that could be temporarily (like related to an ongoing conference) or permanently added to the category options.

I hope you will find this site useful, and feel free to send any feedback to me at @BanjoCam or cameron@asptweets.com

Details for Geeks
  • Entirely programmed in Python 2
  • All scripts run automatically on a (mostly) always-on Raspberry Pi using cron
  • Tweepy used to scrape Twitter API
    • Search run every 15 minutes to get most recent tweets for variety of search terms
  • Tweets saved into local SQLite database using Peewee ORM
  • Flask used to build website
  • Frozen-Flask used to generic static files
    • Generated at 4am daily
  • Modified version of link_preview.py used to generate Link Previews
  • Tweet embedding code grabbed direct from twitter GET API
  • Previews & Embedding HTML cached in database
  • Hosted on Github Pages
